    Intl.MessageFormat

    1 MessageFormat Objects

    1.1 The Intl.MessageFormat Constructor

    The MessageFormat constructor is the %MessageFormat% intrinsic object and a standard built-in property of the Intl object. Behaviour common to all service constructor properties of the Intl object is specified in 9.1.

    1.1.1 Intl.MessageFormat ( source, [ locales [ , options ] ] )

    When the Intl.MessageFormat function is called with arguments source, locales, and options, the following steps are taken:

    1. If NewTarget is undefined, throw a TypeError exception.
    2. Let messageFormat be ? OrdinaryCreateFromConstructor(NewTarget, %MessageFormat.prototype%, « [[InitializedMessageFormat]], [[LocaleMatcher]], [[MessageData]], [[RequestedLocales]], [[Runtime]] »).
    3. Return ? InitializeMessageFormat(messageFormat, source, locales, options).

    1.1.2 InitializeMessageFormat ( messageFormat, source, locales, options )

    The abstract operation InitializeMessageFormat accepts the arguments messageFormat (which must be an object), source, locales, and options. It initializes messageFormat as a MessageFormat object. The following steps are taken:

    1. If source is undefined, throw a TypeError exception.
    2. Let msgData be ? GetMessageData(source).
    3. Let requestedLocales be ? CanonicalizeLocaleList(locales).
    4. Set options to ? GetOptionsObject(options).
    5. Let matcher be ? GetOption(options, "localeMatcher", string, « "lookup", "best fit" », "best fit").
    6. Let userFunctions be ? Get(options, "functions").
    7. Let functions be ? GetMessageFunctions(userFunctions).
    8. Set messageFormat.[[MessageData]] to msgData.
    9. Set messageFormat.[[RequestedLocales]] to requestedLocales.
    10. Set messageFormat.[[LocaleMatcher]] to matcher.
    11. Set messageFormat.[[Functions]] to functions.
    12. Return messageFormat.

    1.2 Properties of the Intl.MessageFormat Constructor

    The Intl.MessageFormat constructor has the following properties:

    1.2.1 Intl.MessageFormat.prototype

    The value of Intl.MessageFormat.prototype is %MessageFormat.prototype%.

    This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

    1.2.2 Internal slots

    Note
    Unlike other Intl formatters, MessageFormat does not have a single list of locales that it supports, as it calls on other formatters as necessary.

    1.3 Properties of the Intl.MessageFormat Prototype Object

    The Intl.MessageFormat prototype object is itself an ordinary object. %MessageFormat.prototype% is not an Intl.MessageFormat instance and does not have an [[InitializedMessageFormat]] internal slot or any of the other internal slots of Intl.MessageFormat instance objects.

    1.3.1 Intl.MessageFormat.prototype.constructor

    The initial value of Intl.MessageFormat.prototype.constructor is %MessageFormat%.

    1.3.2 Intl.MessageFormat.prototype [ @@toStringTag ]

    The initial value of the @@toStringTag property is the String value "Intl.MessageFormat".

    This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }.

    1.3.3 Intl.MessageFormat.prototype.format ( [ values [ , onError ] ] )

    When the format method is called with optional arguments values and onError, the following steps are taken:

    1. Let mf be the this value.
    2. Perform ? RequireInternalSlot(mf, [[InitializedMessageFormat]]).
    3. Let msg be ? ResolveMessage(mf, values, onError).
    4. Let result be an empty String.
    5. For each element mv of msg, do
      1. If mv is a String, then
        1. Let stringValue be mv.
      2. Else,
        1. Let toString be ? Get(mv, "toString").
        2. Let toStringResult be Completion(Call(toString, mv)).
        3. If toStringResult is a normal completion, then
          1. Let stringValue be toStringResult.[[Value]].
        4. Else,
          1. Perform ? HandleMessageFormatError(onError, stringResult).
          2. Let stringValue be MessageFallbackString(mv).
      3. Set result to the string-concatenation of result and stringValue.
    6. Return result.

    1.3.4 Intl.MessageFormat.prototype.formatToParts ( [ values [ , onError ] ] )

    When the formatToParts method is called with optional arguments values and onError, the following steps are taken:

    1. Let mf be the this value.
    2. Perform ? RequireInternalSlot(mf, [[InitializedMessageFormat]]).
    3. Let msg be ? ResolveMessage(mf, values, onError).
    4. Let result be an empty List of Objects.
    5. For each element mv of msg, do
      1. If mv is a String, then
        1. Let textPart be OrdinaryObjectCreate(%Object.prototype%).
        2. Perform ! CreateDataPropertyOrThrow(textPart, "type", "text").
        3. Perform ! CreateDataPropertyOrThrow(textPart, "value", mv).
        4. Append textPart to result.
      2. Else,
        1. Let toParts be ? Get(mv, "toParts").
        2. Let partResult be Completion(Call(toParts, mv)).
        3. If partResult is a normal completion, then
          1. Let parts be partResult.[[Value]].
          2. For each part of parts, append part to result.
        4. Else,
          1. Perform ? HandleMessageFormatError(onError, partResult).
          2. Let fallbackPart be MessageFallbackPart(mv).
          3. Append fallbackPart to result.
    6. Return CreateArrayFromList(result).

    1.3.5 Intl.MessageFormat.prototype.resolvedOptions ( )

    This function provides access to the locale and options computed during initialization of the object.

    1. Let mf be the this value.
    2. Perform ? RequireInternalSlot(mf, [[InitializedMessageFormat]]).
    3. Let options be OrdinaryObjectCreate(%Object.prototype%).
    4. For each row of Table 1, except the header row, in table order, do
      1. Let p be the Property value of the current row.
      2. Let v be the value of mf's internal slot whose name is the Internal Slot value of the current row.
      3. If v is not undefined, then
        1. Perform ! CreateDataPropertyOrThrow(options, p, v).
    5. Return options.

    1.5 Abstract Operations for MessageFormat Objects

    1.5.1 GetMessageData ( source )

    The implementation-defined abstract operation GetMessageData takes argument source (a String).

    It determines the message data representation corresponding to the input source according to the Unicode MessageFormat 2.0 syntax. The message data representation is an Object conforming to the JSON Schema definition of the Unicode MessageFormat 2.0 specification.

    If source contains a syntax or data model error, this operation throws a %SyntaxError%.

    1.5.3 HandleMessageFormatError ( onError, completionRecord )

    The abstract operation HandleMessageFormatError takes arguments onError (a Function or undefined) and completionRecord (a Completion Record). It handles errors during formatting. It performs the following steps when called:

    1. Assert: completionRecord is a Completion Record.
    2. Assert: completionRecord is an abrupt completion.
    3. Let error be completionRecord.[[Value]].
    4. If onError is not undefined, then
      1. Perform ? Call(onError, undefined, « error »).
    5. Else if an appropriate notification mechanism exists, then
      1. An implementation should issue a warning for error.

    1.5.4 MessageFallbackPart ( mv )

    The abstract operation MessageFallbackPart takes argument mv (an Object). It gets the fallback formatted part representation of a MessageValue It performs the following steps when called:

    1. Let result be OrdinaryObjectCreate(%Object.prototype%).
    2. Let source be MessageValueSource(mv).
    3. Perform ! CreateDataPropertyOrThrow(result, "type", "fallback").
    4. Perform ! CreateDataPropertyOrThrow(result, "source", source).
    5. Return result.

    1.5.5 MessageFallbackString ( mv )

    The abstract operation MessageFallbackString takes argument mv (an Object). It gets the fallback formatted string representation of a MessageValue It performs the following steps when called:

    1. Let source be MessageValueSource(mv).
    2. Let result be the string-concatenation of "{", source, and "}".
    3. Return result.

    1.5.6 MessageValueSource ( mv )

    The abstract operation MessageValueSource takes argument mv (an Object). It gets the source string representation of a MessageValue, which should be always available. On any error, return the Unicode replacement character � instead. It performs the following steps when called:

    1. Let source be Completion(Get(mv, "source")).
    2. If source is a normal completion, then
      1. Let str be source.[[Value]].
      2. If str is a String, then
        1. Return str.
    3. Return "\uFFFD".

    1.5.8 ResolveMessage ( mf, values, onError )

    The abstract operation ResolveMessage takes arguments mf (an Object), values (an Object), and onError (a Function or undefined). It resolves a message during formatting into a List of MessageValue Objects. It performs the following steps when called:

    1. Let result be an empty List.
    2. Let pattern be ? SelectPattern(mf, values, onError).
    3. For each element el of pattern,
      1. If el is a String, then
        1. Append el to result.
      2. Else,
        1. Let mv be ResolveExpression(mf, values, onError, el).
        2. Append mv to result.
    4. Return result.

    1.5.9 SelectPattern ( mf, values, onError )

    The abstract operation SelectPattern takes arguments mf (an Object), values (an Object), and onError (a Function or undefined). It selects the pattern to format from the message data model. It performs the following steps when called:

    1. Let msgData be mf.[[MessageData]].
    2. Let msgType be ? Get(msgData, "type").
    3. Assert: msgType is a String.
    4. If msgType is "message", then
      1. Let pattern be ? Get(msgData, "pattern").
    5. Else,
      1. Let selectorsData be ? Get(msgData, "selectors").
      2. Let selectorExpressions be ? CreateListFromArrayLike(selectorsData, « Object »).
      3. Let selectors be an empty List of Object and Null values.
      4. For each element selExp of selectorExpressions,
        1. Let sel be ResolveExpression(mf, values, onError, selExp).
        2. Let selectKeys be ? Get(sel, "selectKeys").
        3. If IsCallable(selectKeys) is true, then
          1. Append sel to selectors.
        4. Else,
          1. Let errorCompletion be ThrowCompletion(TypeError).
          2. Perform ? HandleMessageFormatError(onError, errorCompletion).
          3. Append null to selectors.
      5. Let variantsArray be ? Get(msgData, "variants").
      6. Let variants be ? CreateListFromArrayLike(variantsArray, « Object »).
      7. Let selectedVariant be ? SelectVariant(selectors, variants, onError).
    6. Let patternBody be ? Get(pattern, "body").
    7. Return ? CreateListFromArrayLike(patternBody, « String, Object »).

    1.5.10 SelectVariant ( selectors, variants, onError )

    The implementation-defined abstract operation SelectVariant takes arguments selectors (a List of Object and Null values), variants (a List of Objects), and onError (a Function or undefined).

    It applies the variant selection method defined by the "Resolve Preferences", "Filter Variants", and "Sort Variants" steps of the Pattern Selection section of the Unicode MessageFormat 2.0 specification.

    For the selection, Null values in selectors indicate selectors for which selection will always fail. If any errors are encountered during the selection, the HandleMessageFormatError abstract operation must be called with onError and an abrupt Completion Record containing an Error.

    The value returned by this operation must be an Object matching the JSON Schema definition of a message pattern: https://github.com/unicode-org/message-format-wg/blob/main/spec/data-model/message.json#/$defs/pattern .
